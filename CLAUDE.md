# Learn X - Claude Code 操作指南

**项目类型**: 技术学习资源中心
**最后更新**: 2025-11-19

## ⭐ 项目特点

这是一个**学习资源集合项目**，包含多个独立的学习子模块。每个子模块有自己的学习路径、代码示例和文档。

### 与普通项目的区别
- ✅ 代码主要用于**学习和实验**，不是生产代码
- ✅ 允许多种实现方式并存（对比学习）
- ✅ 文档重点在**解释概念和记录学习心得**
- ✅ 测试不是必须的（除非是学习测试）

## 🎯 我（Claude Code）要做什么？

| 场景 | 应该读什么 | 操作指南 |
|------|-----------|---------|
| 🆕 添加新的学习内容 | 对应模块的 README + 全局 FEATURE_CODE_MAP.md | 1. 在对应模块下创建内容<br>2. 更新模块的 README<br>3. 更新 FEATURE_CODE_MAP.md |
| 📖 查找特定知识点 | FEATURE_CODE_MAP.md → 对应文件 | 使用功能映射快速定位 |
| 🔧 完善某个模块 | 模块的 CLAUDE.md（如果有）+ README | 按照模块的学习路线图补充内容 |
| 📝 整理学习笔记 | 模块的文档结构 | 更新 README 或创建专门的笔记文件 |
| 🤔 理解项目结构 | 本文档 + PROJECT_STATUS.md | 快速了解项目组织方式 |

## 📋 项目规范

### 代码规范（宽松）
- ✅ 代码清晰易懂，有必要的注释
- ✅ 文件命名语义化（描述学习的主题）
- ✅ 相关示例放在一起
- ⏸️ 不强制要求严格的代码规范（学习为主）
- ⏸️ 不强制要求测试覆盖（除非学习测试）

### 文档规范（重要）
- ✅ 每个模块必须有 README.md
- ✅ README 必须包含：
  - 模块目标和学习路径
  - 目录结构说明
  - 如何运行示例代码
  - 学习进度追踪（可选）
- ✅ 示例代码应该有注释解释关键概念
- ✅ 复杂的主题应该有独立的说明文档

### 目录结构规范
```
learn_xxx/
├── README.md              # 模块概览（必须）
├── CLAUDE.md              # AI 操作指南（推荐）
├── 01_topic/              # 按主题或难度组织
│   ├── README.md          # 主题说明
│   ├── example1.xxx       # 示例代码
│   └── example2.xxx
├── 02_topic/
└── resources/             # 学习资源索引（可选）
```

## 🎯 核心工作流

### 添加新的学习内容

**步骤**：
1. 确定内容归属（哪个模块、哪个主题）
2. 在对应目录创建文件
3. 编写代码和注释
4. 更新模块的 README（如果需要）
5. 更新 FEATURE_CODE_MAP.md

**示例**：
```bash
# 假设要在 learn_cpp 添加智能指针示例
cd learn_cpp/02_memory_management/
# 创建 smart_pointers.cpp
# 更新 learn_cpp/02_memory_management/README.md
# 更新根目录的 FEATURE_CODE_MAP.md
```

### 完善某个模块

**步骤**：
1. 阅读模块的 README 和学习路线图
2. 识别缺失的内容或需要改进的部分
3. 按照学习路线图的顺序补充内容
4. 更新文档和进度追踪

### 创建新的学习模块

**步骤**：
1. 在根目录创建 `learn_xxx/` 目录
2. 参考 `learn_cpp` 或 `learn_nextjs` 创建基础结构
3. 创建 README.md，说明模块目标和学习路径
4. 创建 CLAUDE.md（推荐）
5. 更新根目录的 README.md 和 PROJECT_STATUS.md
6. 更新 FEATURE_CODE_MAP.md

## 🚫 禁止事项

### 不要过度规范化
- ❌ 不要要求严格的代码风格（学习项目允许多样化）
- ❌ 不要强制要求测试（除非学习测试技术）
- ❌ 不要过度抽象和封装（保持代码简单易懂）

- ❌ 不要随意修改已有的学习路线图
- ❌ 不要删除对比学习的多种实现方式
- ❌ 不要将简单概念复杂化
