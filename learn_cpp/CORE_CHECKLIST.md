# C++ 核心知识检查清单

> **学习目标**: 掌握 C++ 核心概念，能独立完成小型项目
> **预计时间**: 10-14天 (广度优先学习)
> **毕业标准**: 完成核心实战项目并理解关键代码模式

---

## 📋 阶段 1: 环境和基础 (第1天)

### 环境搭建
- [ ] 安装 C++ 编译器 (g++ 或 clang++)
  - macOS: `xcode-select --install`
  - Linux: `sudo apt install g++`
- [ ] 验证安装: `g++ --version`
- [ ] 成功编译并运行 Hello World
  ```bash
  g++ -std=c++17 examples/01_hello.cpp -o hello
  ./hello
  ```

### 核心理解点
- [ ] 理解编译过程: 源码 → 编译 → 可执行文件
- [ ] 知道 `.h` 头文件和 `.cpp` 源文件的区别
- [ ] 能够使用基本的 `std::cout` 和 `std::cin`

**验收**: 能够编写一个简单的输入输出程序

---

## 🎯 阶段 2: 核心概念 (第2-5天)

### 2.1 内存管理基础 ⭐⭐⭐ (最重要!)
- [ ] **指针**: 理解指针就是内存地址
  - [ ] 能够声明和使用指针: `int* ptr = &value;`
  - [ ] 理解解引用: `*ptr` 访问指针指向的值
  - [ ] 运行 `examples/07_pointers.cpp`

- [ ] **引用**: 理解引用是变量的别名
  - [ ] 知道引用和指针的区别 (引用不能为空，不能重新绑定)
  - [ ] 运行 `examples/08_references.cpp`

- [ ] **动态内存**: 理解栈和堆的区别
  - [ ] 掌握 `new/delete` 的使用
  - [ ] 知道内存泄漏的概念
  - [ ] 运行 `examples/10_dynamic_memory.cpp`

- [ ] **智能指针**: 现代C++的内存管理
  - [ ] 理解 `std::unique_ptr` (独占所有权)
  - [ ] 理解 `std::shared_ptr` (共享所有权)
  - [ ] 运行 `examples/11_smart_pointers.cpp`

**验收**: 能够解释什么是内存泄漏，并知道如何使用智能指针避免

### 2.2 类和对象 ⭐⭐
- [ ] **类的基本结构**
  - [ ] 能够定义类: 成员变量、成员函数
  - [ ] 理解访问控制: `public`, `private`, `protected`

- [ ] **构造和析构**
  - [ ] 理解构造函数的作用
  - [ ] 理解析构函数在对象销毁时自动调用
  - [ ] 理解 RAII 原则 (资源获取即初始化)

- [ ] **拷贝控制** (简单了解即可)
  - [ ] 知道拷贝构造函数的存在
  - [ ] 知道深拷贝和浅拷贝的区别

**验收**: 能够编写一个包含构造函数和析构函数的简单类

### 2.3 标准库基础 ⭐⭐
- [ ] **std::string**: 字符串操作
  - [ ] 掌握基本操作: `length()`, `substr()`, `find()`

- [ ] **std::vector**: 动态数组
  - [ ] 掌握基本操作: `push_back()`, `size()`, `[]` 访问
  - [ ] 理解 vector 自动管理内存

- [ ] **std::map**: 键值对容器
  - [ ] 掌握基本操作: `insert()`, `find()`, `[]` 访问

**验收**: 能够使用 vector 和 map 完成简单的数据处理任务

### 2.4 函数和模板 ⭐
- [ ] **函数重载**: 同名函数不同参数
- [ ] **函数模板**: 泛型编程基础
  ```cpp
  template<typename T>
  T max(T a, T b) { return a > b ? a : b; }
  ```
- [ ] **Lambda 表达式**: 匿名函数
  ```cpp
  auto add = [](int a, int b) { return a + b; };
  ```

**验收**: 能够编写简单的模板函数

## 🐛 阶段 4: 常见问题处理 (持续)

### 问题 1: 段错误 (Segmentation Fault)
- [ ] 遇到过这个错误
- [ ] 知道通常是访问了无效内存地址导致
- [ ] 掌握调试方法:
  - 使用 `gdb` 调试: `gdb ./program`
  - 检查指针是否为空
  - 检查数组越界

### 问题 2: 内存泄漏
- [ ] 理解什么是内存泄漏
- [ ] 知道如何检测: `valgrind --leak-check=full ./program`
- [ ] 掌握预防方法:
  - 优先使用智能指针
  - new/delete 成对使用
  - 使用 RAII 原则

### 问题 3: 编译错误
- [ ] 遇到过 "undefined reference" 错误
  - 原因: 链接阶段找不到函数实现
  - 解决: 确保所有 .cpp 文件都被编译

- [ ] 遇到过 "multiple definition" 错误
  - 原因: 同一个函数在多个文件中定义
  - 解决: 使用头文件保护 `#ifndef`, `#define`, `#endif`

### 问题 4: 运行时错误
- [ ] vector 下标越界
  - 使用 `.at()` 代替 `[]` (会抛出异常)

- [ ] 空指针访问
  - 使用前检查: `if (ptr != nullptr)`
